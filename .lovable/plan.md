
# ربط تلغرام بالتطبيق - Telegram Webhook Integration

## الفكرة
إنشاء بوت تلغرام يستقبل الرسائل من قنواتك على تلغرام وينشرها تلقائياً في التطبيق. عند نشر رسالة تحتوي على هاشتاغ `#نشر` أو `#publish` في قناة تلغرام، يتم إدراجها تلقائياً في التطبيق.

## كيف يعمل النظام

```text
+------------------+       +---------------------+       +------------------+
|  قناة تلغرام     | ----> | Telegram Bot API     | ----> | Edge Function    |
|  (VIP/عامة/أخبار)|       | (Webhook)            |       | telegram-webhook |
+------------------+       +---------------------+       +------------------+
                                                                  |
                                                    +-------------+-------------+
                                                    |             |             |
                                              +-----v----+  +----v-----+  +----v-----+
                                              | signals   |  | signals  |  | articles |
                                              | (free)    |  | (vip)    |  | (news)   |
                                              +----------+  +----------+  +----------+
```

## التوزيع حسب القناة
- **القناة العامة** (Chat ID محدد) --> جدول `signals` بصلاحية `free`
- **قناة VIP** (Chat ID محدد) --> جدول `signals` بصلاحية `vip`
- **قناة الأخبار** (Chat ID محدد) --> جدول `articles`

## المتطلبات المسبقة
1. إنشاء بوت تلغرام عبر @BotFather والحصول على Bot Token
2. إضافة البوت كمشرف (Admin) في القنوات الثلاث
3. الحصول على Chat ID لكل قناة

## خطوات التنفيذ

### 1. إعداد الأسرار (Secrets)
تخزين المفاتيح التالية بشكل آمن:
- `TELEGRAM_BOT_TOKEN` - توكن البوت من BotFather
- `TELEGRAM_PUBLIC_CHAT_ID` - معرف القناة العامة
- `TELEGRAM_VIP_CHAT_ID` - معرف قناة VIP
- `TELEGRAM_NEWS_CHAT_ID` - معرف قناة الأخبار
- `TELEGRAM_WEBHOOK_SECRET` - كلمة سر عشوائية لتأمين الاتصال

### 2. إنشاء Edge Function: `telegram-webhook`
دالة خلفية تستقبل تحديثات تلغرام وتعالجها:

- تتحقق من صحة الطلب عبر `secret_token`
- تقرأ الرسالة وتفحص وجود هاشتاغ `#نشر` أو `#publish`
- تحدد القناة المصدر بناءً على `chat.id`
- تستخرج النصوص والصور من الرسالة
- تدرج المحتوى في الجدول المناسب (`signals` أو `articles`)
- تتعامل مع الصور عبر تحميلها من Telegram API وحفظها في التخزين

### 3. إنشاء Edge Function: `setup-telegram-webhook`
دالة مساعدة لتسجيل عنوان Webhook لدى تلغرام تلقائياً (تُستدعى مرة واحدة فقط).

### 4. تحديث إعدادات config.toml
إضافة الدوال الجديدة مع `verify_jwt = false` لأن تلغرام يرسل الطلبات مباشرة بدون مصادقة JWT.

### 5. إضافة لوحة إعدارة في CMS (اختياري)
إضافة قسم بسيط في لوحة الإدارة لعرض حالة الربط مع تلغرام وزر لإعادة تسجيل الـ Webhook.

## التفاصيل التقنية

### بنية Edge Function الرئيسية (`telegram-webhook/index.ts`)

```text
1. استقبال POST من Telegram Bot API
2. التحقق من secret_token في الهيدر
3. تحليل نوع التحديث (message / channel_post)
4. فحص وجود #نشر أو #publish في النص
5. تحديد الوجهة:
   - chat_id == PUBLIC  --> signals (visibility: free)
   - chat_id == VIP     --> signals (visibility: vip)
   - chat_id == NEWS    --> articles
6. معالجة الصور (إن وجدت):
   - جلب الملف من Telegram API
   - رفعه إلى Storage bucket المناسب
7. إدراج السجل في قاعدة البيانات
8. إرجاع 200 OK
```

### معالجة أنواع المحتوى
- **نص فقط**: يُحفظ مباشرة كمحتوى
- **صورة + نص (caption)**: يُحفظ النص + رابط الصورة بعد رفعها
- **عدة صور (media group)**: يتم تجميعها وحفظها كمصفوفة مرفقات

### الأمان
- التحقق من `X-Telegram-Bot-Api-Secret-Token` في كل طلب
- استخدام Service Role Key فقط داخل الدالة الخلفية
- عدم قبول أي طلبات من مصادر غير موثوقة

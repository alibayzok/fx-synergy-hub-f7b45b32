

# خطة النشر المزدوج: Vercel (ويب) + Google Play (APK)

## الهدف
تجهيز المشروع للنشر على منصتين معاً:
1. **Vercel** - نسخة الويب (PWA)
2. **Google Play** - تطبيق أندرويد مستقل (APK/AAB)

---

## المرحلة 1: تجهيز Vercel (نسخة الويب)

### 1.1 إنشاء ملف `vercel.json`
ملف توجيه SPA في جذر المشروع لضمان عمل التنقل بين الصفحات بشكل صحيح.

### 1.2 خطوات النشر على Vercel (يدوية)
1. تصدير المشروع إلى GitHub عبر زر "Export to GitHub"
2. ربط المستودع بحساب Vercel
3. إضافة متغيرات البيئة الثلاثة في إعدادات Vercel
4. النشر تلقائي مع كل تحديث

---

## المرحلة 2: تجهيز APK لـ Google Play

### 2.1 تحديث `capacitor.config.ts`
- تغيير `appId` إلى `com.assassinfx.app` (للإنتاج)
- إضافة تعليق واضح لحذف قسم `server` عند بناء APK

### 2.2 إضافة صفحة سياسة الخصوصية
صفحة `/privacy-policy` مطلوبة من Google Play، تتضمن:
- جمع البيانات واستخدامها
- الأمان والتشفير
- حقوق المستخدم

### 2.3 خطوات بناء APK (يدوية)
1. تصدير المشروع إلى GitHub
2. `npm install` ثم `npm run build`
3. حذف قسم `server` من `capacitor.config.ts`
4. `npx cap sync android`
5. فتح Android Studio وبناء AAB موقّع
6. رفعه على Google Play Console

---

## التفاصيل التقنية

### الملفات المطلوب إنشاؤها
| الملف | الغرض |
|---|---|

### الملفات المطلوب تعديلها
| الملف | التعديل |
|---|---|
| `capacitor.config.ts` | تحديث appId للإنتاج |
| `src/App.tsx` | إضافة مسار صفحة سياسة الخصوصية |

### ملاحظات مهمة
- نسخة Vercel ونسخة APK تستخدمان نفس قاعدة البيانات
- بناء APK يتطلب Android Studio على جهازك
- حساب Google Play يكلف 25 دولار (مرة واحدة)
- يجب حذف قسم `server` فقط عند بناء APK النهائي، وليس أثناء التطوير
